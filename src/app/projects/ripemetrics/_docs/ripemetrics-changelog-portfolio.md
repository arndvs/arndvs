# RipeMetrics Changelog — Portfolio Report

> Deep codebase audit of `ripemetrics-ripe-changelog-0d3ef5ce7aa2` — a public-facing product changelog built on Next.js + MDX.

---

## Executive Summary

RipeMetrics Changelog is a **single-page product changelog app** built on **Next.js 14 (App Router) + MDX**. Based on the Tailwind UI "Commit" template, it renders all changelog entries from a single `page.mdx` file into a timeline with a fixed sidebar. The app was launched in October 2024 and only captured **5 entries over 6 weeks** before updates stopped — despite years of prior product development that were never backfilled.

## Architecture

### Single-File-as-Database

The entire changelog lives in one file — `src/app/page.mdx`. No CMS, no database, no API. Each entry is separated by an `<hr>`, and the MDX pipeline splits them into `<article>` elements at build time.

Entry format:

```mdx
---

## Entry Title {{ date: '2024-10-24T00:00Z' }}

Description paragraph...

### <SparkleIcon /> Improvements

- Bullet point items

### <NewFeatureIcon /> New Features

- Bullet point items

### Bug Fixes

- Bullet point items
```

### MDX Pipeline (3 Stages)

1. **Remark** — `mdx-annotations`, `remark-gfm`, `remark-unwrap-images`
2. **Rehype** — `mdx-annotations`, `rehype-slug`, `rehype-autolink-headings`, custom Shiki syntax highlighting, **`remark-rehype-wrap`** (the key plugin that splits on `<hr>` and wraps into `<article>` with date metadata)
3. **Recma** — `mdx-annotations`, `recma-import-images`

### Layout Pattern

Fixed left sidebar (brand intro, newsletter form, icon links, tagline) + scrolling right pane of timeline articles. `ResizeObserver` snaps article heights to multiples of 8px for consistent timeline alignment.

## Notable Technical Patterns

1. **Single-file-as-database** — Entire changelog lives in one `page.mdx`, split into articles by the MDX pipeline at build time. Simple but effective for a low-volume changelog.

2. **Runtime RSS self-scraping** — The `/feed.xml` route handler fetches the app's own rendered homepage and parses it with Cheerio to generate RSS. Clever but fragile — and still contains un-customized template metadata (author: "Joe Davola", description: "Open-source Git client for macOS minimalists").

3. **ResizeObserver height snapping** — Articles snap to 8px grid multiples for consistent timeline dot alignment.

4. **MDX annotations** — Date metadata is encoded inline in h2 headings as `{{ date: '...' }}` annotations, parsed by the `mdx-annotations` plugin.

## Key Takeaways for Portfolio

1. **Product communication tool** — Public-facing changelog for customer transparency and engagement
2. **MDX-powered content pipeline** — Custom remark/rehype/recma plugin chain for structured content from markdown
3. **Template customization** — Tailwind UI "Commit" template adapted with RipeMetrics branding, dark mode, and PostHog analytics
4. **Newsletter integration** — Direct API integration with the main RipeMetrics marketing site
5. **Represents a gap** — Only 6 weeks of a multi-year product journey were captured, making it a cautionary example of changelog discipline
